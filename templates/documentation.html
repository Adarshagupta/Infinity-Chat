API Keys

API keys are essential for authenticating your requests to the Chatcat API. Here's important information about managing your API keys:

Obtaining an API Key

  1. Log in to your Chatcat dashboard
  2. Navigate to the "API Keys" section
  3. Click on "Generate New API Key"
  4. Copy and securely store your new API key

Refreshing API Keys

Note: There is currently an issue with the API key refresh functionality. Our team is working on resolving this. In the meantime, if you need to update your API key, please contact our support team at support@chatcat.com.

Once the issue is resolved, you will be able to refresh your API key by following these steps:

  1. Go to your Chatcat dashboard
  2. Navigate to the "API Keys" section
  3. Find the API key you want to refresh
  4. Click on the "Refresh" button next to the key
  5. Confirm the action when prompted

Remember to update your applications with the new API key after refreshing.

API Reference

Chat Endpoint

POST /chat
Content-Type: application/json

{
    "input": "User message",
    "api_key": "your_api_key"
}

This endpoint allows you to send user messages to the chatbot and receive AI-generated responses.

Process URL Endpoint

POST /dashboard/home/process_url
Content-Type: application/json

{
    "url": "https://example.com",
    "llm": "together"  // or "openai"
}

Use this endpoint to process a website URL and create a new chatbot.

Error Handling

Our API uses conventional HTTP response codes to indicate success or failure of requests. In case of errors, you may receive one of the following:

  • 400 Bad Request - Invalid parameters
  • 401 Unauthorized - Invalid API key
  • 404 Not Found - Resource doesn't exist
  • 429 Too Many Requests - Rate limit exceeded
  • 500 Internal Server Error - Server-side issue

If you encounter persistent errors, especially related to API key authentication, please contact our support team.
